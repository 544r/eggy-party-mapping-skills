# 蛋仔派对地图制作 - 完整教程

## 目标

从零开始，手把手教你制作一张完整的可玩地图。

---

## 第一层：线性引导——打造你的第一个可玩地图

对于完全没有创作经验的新手，最有效的学习方式是**立即动手实践**。本章节将带你从打开游戏开始，直到成功发布第一张地图。

---

### 第一步：创建新地图

1. 打开蛋仔派对游戏
2. 进入**工坊**（乐园 → 蛋仔工坊）
3. 点击**新建地图**
4. 选择地图尺寸（初学者建议选小尺寸）

> 进入编辑器后，你会看到一个三维空间，包含X、Y、Z坐标轴和网格辅助线。

---

### 第二步：放置出生点

出生点是玩家开始游戏时出现的位置。

**路径：** 组件库 → 机关 → 逻辑 → 出生点

**操作：**
1. 打开左侧组件库
2. 点击「机关」分类
3. 选择「逻辑」
4. 找到「出生点」，拖拽到场景中

**调整：** 选中出生点后，用移动工具调整位置。可以在右侧属性面板设置精确坐标。

---

### 第三步：创建平台

玩家需要有地方站立和行走。

**路径：** 组件库 → 结构 → 基础形状

**操作：**
1. 组件库 → 结构
2. 选择「基础形状」
3. 找到「平台」（或其他形状）
4. 拖拽到场景中，放在出生点附近

**调整平台：**
- 选中平台后，底部出现工具栏
- **移动**：沿X/Y/Z轴调整位置
- **缩放**：调整长宽高
- **旋转**：调整角度

---

### 第四步：放置胜利区域

当玩家到达这里时，游戏判定为胜利。

**路径：** 组件库 → 机关 → 逻辑 → 胜利区域

**操作：**
1. 组件库 → 机关 → 逻辑
2. 找到「胜利区域」
3. 拖拽到平台末端

**调整：** 确保胜利区域位置是玩家能够走到的地方。

---

### 第五步：试玩测试

在发布前，必须测试地图是否可玩。

**路径：** 底部中央 → 试玩按钮

**操作：**
1. 点击底部中央的「试玩」按钮
2. 选择模式：
   - **普通模式**：正常游戏测试
   - **沙盒模式**：可暂停，边玩边编辑
3. 检查：
   - 玩家是否在出生点出现
   - 能否正常行走和跳跃
   - 到达胜利区域是否显示胜利

**发现问题：** 返回编辑器调整组件位置，重新测试。

---

### 第六步：发布地图

测试成功后可以发布。

**路径：** 设置 → 发布地图

**操作：**
1. 点击顶部「设置」按钮
2. 点击「发布地图」
3. 如果提示"需要先到达终点"，先完成试玩
4. 填写信息（标题、简介、封面、标签）
5. 提交审核

---

## 第二层：模块深化

当第一层的最小地图成功后，可以开始学习各个功能模块。

---

### 模块一：地形与平台构建

#### 地形编辑器

地形组件可以创造出高低起伏的自然地貌。

**路径：** 组件库 → 结构 → 地形组件

**笔刷工具：**

| 笔刷 | 功能 |
|------|------|
| 高度笔刷 | 绘制地形高低起伏，生成物理碰撞 |
| 平滑笔刷 | 打磨棱角，减小高度差 |
| 整平笔刷 | 创造平坦地形 |
| 材质笔刷 | 更改地形纹理样式 |
| 染色笔刷 | 叠加局部染色效果 |
| 装饰笔刷 | 绘制植被等装饰物 |
| 流体笔刷 | 绘制流水景观 |

> 每种笔刷都可以调节笔刷大小和力度。

#### 特殊地面组件

部分组件有特殊物理效果。

**传送带：** 机关 → 运动类 → 传送带
- 会持续推动玩家移动

**冰面：** 机关 → 运动类
- 增加惯性，制动困难

**弹簧/推进装置：** 机关 → 运动类
- 将玩家弹射到远处或高处

---

### 模块二：机关与互动元素

机关让地图"活起来"，是玩家与关卡交互的核心。

#### 机关分类

**运动类机关：** 机关 → 运动类
- 门与升降梯
- 弹簧与推进装置
- 传送带

**战斗类机关：** 机关 → 战斗类
- 炮台与发射器
- 陷阱与尖刺
- 火焰与电击网

**逻辑类机关：** 机关 → 逻辑
- 出生点、胜利区域
- 逻辑开关（信号处理）
- 音乐切换器

#### 机关联动

将多个机关组合，形成连锁反应。

**示例：** 踩压力板 → 打开门 → 启动炮台

1. 放置压力板（机关 → 逻辑）
2. 放置门（机关 → 运动类）
3. 通过蛋码连接：压力板被触发 → 门打开

---

### 模块三：逻辑组件与蛋码

蛋码是蛋仔工坊的图形化编程工具。

**入口：** 更多 → 蛋码

#### 触发器ECA结构

- **E (Event)**：触发条件
- **C (Condition)**：判断逻辑
- **A (Action)**：执行结果

#### 逻辑开关

工坊使用「逻辑开关」组件实现逻辑运算，不使用"与门/或门"等术语。

**逻辑开关**支持：
- 与运算：所有接入开关为开时才输出
- 或运算：任一接入开关为开即输出

**延时器：** 接收信号后延迟一段时间再发送

#### 积木类型

| 类型 | 用途 |
|------|------|
| 事件积木 | 定义触发条件 |
| 动作积木 | 执行操作 |
| 控制积木 | 循环/条件分支 |
| 条件积木 | 判断逻辑 |
| 取值积木 | 获取数据 |

#### 调试功能

- **运行日志**：显示错误信息，点击可跳转问题积木
- **添加断点**：长按积木 → 添加断点，暂停调试

---

### 模块四：装饰、测试与发布

#### 装饰与氛围

**道具分类：** 组件库 → 道具
- 收集类：道具箱、金币
- 装饰类：植被、建筑、特效

**设计原则：**
- 主题一致：装饰风格要统一
- 层次感：前景、中景、背景合理搭配
- 叙事性：通过装饰讲述故事

#### 测试清单

发布前检查：

| 检查项 | 内容 |
|--------|------|
| 功能完整性 | 所有机关是否正常工作 |
| 稳定性 | 有无卡顿、穿模、崩溃 |
| 平衡性 | 难度是否适中 |
| 引导清晰度 | 玩家能否理解目标 |
| 胜利条件 | 能否正常到达胜利区域 |

#### 发布流程

1. 确认所有信息填写完整
2. 检查地图占用值是否超标
3. 提交审核
4. 等待官方审核通过

## 常见问题

**Q：玩家出生后掉下去了**
→ 检查出生点是否在安全位置，下方是否有阻挡

**Q：走到胜利区域没反应**
→ 检查胜利区域位置是否玩家可达，区域大小是否足够

**Q：机关不生效**
→ 检查是否正确连接，用试玩模式测试逻辑链

**Q：地形编辑在哪里**
→ 组件库 → 结构 → 地形组件，不是道具分类
